# admin-ui 模块

## 职责
- 管理台前端展示与操作
- 覆盖站点、模型、令牌、日志、面板与设置

## 接口定义
- 调用 `/api/*` 管理接口
- 登录后使用 Bearer token

## 行为规范
- Vite 构建静态文件
- 构建产物 `apps/ui/dist` 由 Worker Static Assets 托管
- 使用 `hono/jsx/dom` + TSX 渲染管理台入口
- 入口文件为 `apps/ui/src/App.tsx`（`main.tsx` 已移除）
- 按 `core/`（类型/常量/API）、`features/`（业务视图 + 布局/登录）组织前端模块
- 样式基于 Tailwind v4（结合全局主题变量）
- 数据面板提供 Area Chart 趋势图与时间范围筛选（含“全部”、近 7/30/90 天、近一年、自定义）
- 统计颗粒（按日/周/月）放在趋势图标题区，与自定义列并排
- 数据面板时间范围与统计颗粒本地记忆，优先使用上次选择
- 数据面板筛选条紧凑化，渠道/令牌/模型收纳为高级筛选（渠道/令牌为搜索多选）
- 数据面板排行模块使用等分 Tabs 切换（模型/渠道/令牌）
- 前端状态集中在单页
- 站点管理采用列表视图，支持分页（默认 10 条），新增/编辑使用弹窗，保留启禁/连通测试操作
- 站点管理支持名称/URL 搜索与列头排序，搜索/排序后再分页
- 站点类型支持 new-api / done-hub / subapi / openai / Anthropic / gemini，类型驱动表单字段
- 站点表单提供系统令牌、userid、自动签到开关与外部签到地址（new-api 默认关闭）
- 站点支持多调用令牌（备注 + 令牌值）
- openai/Anthropic/gemini 站点支持 base_url 为空时自动填充官方地址
- done-hub 站点仅使用 base_url，不再维护多地址输入
- 站点列表将“测试”统一替换为“验证”，并展示最近一次验证结论
- 站点新增/修改完成后自动触发站点验证并展示阶段化结果
- 批量操作提供“批量验证”和“评估恢复”，报告按结论与失败原因展示
- 单站点验证弹窗展示 `connectivity / capability / service / recovery` 四阶段卡片与建议动作
- 使用日志支持服务端分页（默认 50 条）与固定高度滚动，并展示令牌、输入/输出 tokens、首 token 延迟、流式与推理强度
- 使用日志支持按日期/渠道/令牌/模型/状态码筛选并与分页联动（渠道/令牌/模型/状态均为搜索多选）
- 使用日志中的模型展示统一以 `canonical_model` 为主，同时补充 `request_model_raw / upstream_model_raw` 便于排查请求名与真实上游模型名差异
- 使用日志状态列仅展示上游状态码，缺失时显示“未知”，不做 OK/Error 语义化
- 错误详情弹窗展示元信息、策略动作/命中键/查找键/归一错误码，并保留完整错误摘要与原始诊断元数据
- 令牌管理采用列表视图（默认每页 10 条），生成/编辑使用弹窗表单，支持过期时间（北京时间）与允许渠道（默认全开）并默认复制，列表展示创建时间、前缀、过期时间与渠道限制摘要
- 站点管理内置一键签到入口，展示今日签到结果（基于后台写回的 `last_checkin_*`）
- 系统设置增加定时签到开关与时间（中国时间）
- 系统设置将“禁用渠道自动抽测恢复”改为“已禁用站点自动恢复评估”
- 系统设置提供模型失败冷却分钟数配置
- 系统设置提供运行时配置可编辑（上游超时、流式解析模式/上限、队列开关/日限额/直写比例）
- 系统设置将运行时配置拆分为“解析与请求 / 队列设置”，队列绑定/实际生效/使用数量以内联只读项展示
- 系统设置提供缓存开关与 TTL（面板/日志/模型/鉴权/渠道/调用令牌/设置）配置
- 系统设置展示缓存版本号并提供刷新入口
- 列表分页页脚统一使用页码导航 + 每页条数选择，并显示总页数（无数据时显示 0 页）
- 每页条数选择与列显示偏好本地持久化
- 页面布局拆分为“标题区 + 筛选区 + 数据区”，避免单一大卡片堆叠
- 数据表支持列自定义（本地持久化）
- 模型广场支持搜索、渠道筛选与本地分页
- 模型广场为统一模型聚合行，并展示该统一模型命中的实际别名列表（`raw_ids`）
- 价格中心在页面内提供 USD/CNY 货币切换入口，切换后复用系统设置保存链路并刷新当前价格列表
- 价格中心展示最近一次价格同步摘要，支持查看逐来源成功/失败明细与失败原因
- 价格中心刷新价格设置时只回填未改动字段，不能覆盖系统设置页尚未保存的本地草稿
- Token 仅本地保存，不回显
- 移动端使用顶部栏 + 抽屉导航，侧边栏在桌面端保持固定
- 渠道/令牌列表在移动端以卡片形式展示，操作区改为网格按钮排列
- 数据面板、模型广场、使用日志表格在移动端支持横向滚动与字号收紧
- 全局通知采用右上角 Toast 展示，支持关闭与进度条自动消退
- 删除类操作必须经过确认弹窗
- 主要异步操作提供 loading 与按钮禁用态
- 弹窗支持全屏遮罩/ESC 关闭，不强制自动聚焦首字段
- 空状态提供清晰 CTA 引导下一步
- 全局字体使用系统字体栈，背景包含冷色光晕与轻纹理

## 依赖关系
- `auth` / `channels` / `models` / `tokens` / `usage` / `dashboard` / `settings` 模块
- `sites` 模块
